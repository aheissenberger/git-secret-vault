Implemented NFR-018: GitHub Actions Homebrew tap automation on push to main.

Changes:
- `.github/workflows/release.yml`: added `push: branches: [main]` trigger; added `Determine version` step to `release` job; added `publish-homebrew` job that downloads build artifacts, computes per-arch SHA256 checksums, updates the formula, and pushes to the tap repo via `gh api`.
- `packaging/homebrew/git-secret-vault.rb`: new Homebrew formula with `on_macos`/`on_linux` blocks for aarch64-apple-darwin, x86_64-apple-darwin, x86_64-unknown-linux-gnu.
- `scripts/update-homebrew-formula.py`: Python script to patch the formula version and sha256 fields in-place.
- `docs/homebrew-tap.md`: setup and usage documentation for the tap.

Acceptance criteria satisfied:
1. Push to main triggers release automation (no manual tag required).
2. Build matrix produces arch-specific binaries; SHA256s computed per architecture.
3. `publish-homebrew` job pushes updated formula to tap repo.
4. Re-run behavior is guarded to avoid corrupting tap state.
