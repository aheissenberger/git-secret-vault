## Summary

Implemented `--json` flag for the `doctor` command (`src/cli/doctor.rs`).

Each check now carries a machine-readable `name` field. When `--json` is
passed, output is a JSON object `{ "ok": bool, "checks": [...] }` with per-check
`name`, `ok`, and `message` fields. Default text output is unchanged.

Created `docs/exit-codes.md` documenting the exit-code contract: all errors
(including dirty vault, stale vault, and failed doctor checks) use exit code 1.

Updated `spec/requirements/FR-016.md` status to Done and
`spec/TRACE.md` FR-016 row with implementation pointers.
