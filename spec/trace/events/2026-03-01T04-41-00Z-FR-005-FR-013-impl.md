## Changes

- Added `get_password_with_keyring(from_stdin, vault_uuid, require_keyring, prompt)` implementing the full password source precedence: stdin → VAULT_PASSWORD env var → system keyring (via UUID) → interactive prompt.

- Added `--index`, `--no-keyring`, and `--require-keyring` flags to `UnlockArgs`.
- `run()` now attempts keyring lookup via the vault UUID from the index file.

### src/cli/init.rs
- Added `--no-keyring` (accepted for script consistency, no-op) and `--require-keyring` (returns error since no UUID exists at init time) flags to `InitArgs`.

### src/cli/passwd.rs
- After successful password rotation, automatically updates any existing keyring credential for the vault UUID with the new password (stale-credential refresh).
