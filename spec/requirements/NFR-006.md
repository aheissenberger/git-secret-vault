## Summary

Release process should provide signed artifacts and checksums.

## Acceptance Criteria

- Build pipeline produces signed release assets with verifiable checksums.

## Verification

- Validate signature and checksum generation in release workflow.

## Notes

Maps directly to NFR-REL-1.

## Implementation Evidence

- macOS release binaries: signed with Developer ID certificate via GitHub Actions secrets.
- All release binaries (all platforms): signed with cosign keyless signing using GitHub OIDC.
  - `.sigstore.json` bundle published alongside each binary in GitHub Releases.
  - Signing job: `sign-artifacts` in `.github/workflows/release.yml`.
  - Verification: `cosign verify-blob --bundle <binary>.sigstore.json --certificate-identity-regexp "https://github.com/OWNER/git-secret-vault/.*" --certificate-oidc-issuer "https://token.actions.githubusercontent.com" <binary>`.
