## Summary

ZIP is the interchange export format only; authoritative vault storage is the dual-profile blob/index structure. Standard unzip tools remain usable via explicit export.

## Acceptance Criteria

- `vault zip-export` produces an AES-256 AE-2 ZIP compatible with standard unzip tools.
- The authoritative vault is never stored as a ZIP file; ZIP is produced only on explicit export (`vault zip-export`).
- `compat check` documents ZIP interchange limitations and directs users to `zip-export` for external tool compatibility.
- Standard zip/unzip binaries are not required for normal vault operation (init, lock, unlock, status).

## Verification

- Validate that `vault zip-export` output is openable by documented external unzip tools.
- Validate that vault init/lock/unlock operates successfully without any zip binary present on the system.

## Notes

ZIP interoperability is preserved as an export/import facility (see FR-034). The authoritative format is the blob store and event log.
