## Summary

Support secure lifecycle operations for entry removal, password rotation, and cross-platform keyring management.

## Acceptance Criteria

- `rm` removes vault/manifest entries with pattern support and optional local deletion; operations are atomic.
- `passwd` re-encrypts safely and handles stale keyring credentials with refresh/update flow.
- `passwd --rotate` emits a team-facing rotation checklist.
- `keyring save|status|delete|list|purge` is implemented with vault UUID scoping and optional repo fingerprinting.
- Keyring modes include `--no-keyring`, `--require-keyring`, and optional app-level TTL support.

## Verification

- Validate end-to-end password rotation and keyring stale-credential handling.
- Validate keyring lifecycle operations on macOS/Linux/Windows.

## Notes

Keyring support is mandatory cross-platform, not optional product scope.
