## Summary

`unlock` safely restores selected secrets with explicit conflict policies and atomic per-file writes.

## Acceptance Criteria

- `unlock` restores selected entries or all tracked entries when no path is provided.
- Conflicts are detected when differing local files exist.
- Interactive conflict resolution supports keep-local, overwrite, keep-both, skip, and editor merge for text files.
- Non-interactive conflict flags (`--force`, `--keep-local`, `--keep-both`, `--no-prompt`) are implemented.
- Each restored file uses atomic write semantics.

## Verification

- Run conflict matrix tests for interactive and non-interactive policies.
- Validate atomicity and rollback behavior when errors occur mid-unlock.

## Notes

Conflict UX must minimize accidental data loss while enabling CI-safe non-interactive behavior.
