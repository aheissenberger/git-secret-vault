## Summary

Optimize CLI automation ergonomics for AI agents and non-interactive orchestrators.

## Acceptance Criteria

- Commands used by automation expose deterministic machine-readable output (`--json`) with stable key names and deterministic ordering.
- Automation mode is non-interactive: prompts are disabled when explicit non-interactive flags are provided and commands fail fast with actionable errors.
- Output profile for agent usage supports no ANSI formatting and no pager/editor side effects.
- Exit codes are stable and documented for success, validation failure, auth failure, and operational failure classes.
- Secrets are never emitted in machine-readable or human-readable output.

## Verification

- Run command matrix in non-interactive mode and confirm no prompt blocks.
- Validate JSON output stability across repeated runs for unchanged state.
- Verify exit-code contract for representative failure classes.

## Notes

Recommended pattern for AI-agent usage:

- Always use explicit paths and flags.
- Prefer `--json` + non-interactive password input (`--password-stdin`) in automation.
- Treat stdout as data channel and stderr as diagnostics channel.
- Make each command idempotent where feasible and safe for retries.
