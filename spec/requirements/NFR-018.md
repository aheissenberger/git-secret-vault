## Summary

Create a GitHub Actions release workflow that publishes GitSecretVault to a GitHub Homebrew tap on direct pushes, with architecture-specific release outputs.

## Acceptance Criteria

- A push to the repository default branch triggers release automation without requiring a manual tag step.
- Release automation produces architecture-specific artifacts needed for Homebrew bottle distribution.
- The workflow updates or publishes to a GitHub-hosted Homebrew tap with per-architecture metadata.
- Re-running the workflow for the same commit is safe and does not corrupt tap state.

## Verification

- Validate workflow trigger behavior on direct pushes to the default branch.
- Validate per-architecture release artifacts are produced and mapped in Homebrew tap metadata.
- Validate tap install succeeds for each supported architecture using the published release outputs.

## Notes

This requirement defines release and packaging automation behavior only; implementation details (single workflow vs split workflows, tap repository strategy, and supported architecture matrix) are decided during implementation.
