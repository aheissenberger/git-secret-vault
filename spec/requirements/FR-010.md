## Summary

`lock` encrypts tracked content into the vault with deterministic/atomic behavior and optional plaintext cleanup.

## Acceptance Criteria

- `lock` encrypts selected paths and updates encrypted manifest metadata.
- No-argument `lock` processes all tracked entries.
- Pattern handling is supported with consistent cross-platform behavior, including Windows.
- `--remove` deletes plaintext after successful encryption.
- `--check` performs drift validation only and exits non-zero on mismatch.
- Vault rewrite path is atomic.

## Verification

- Validate selective/no-arg lock modes, pattern handling, and `--check` semantics.
- Validate no plaintext deletion occurs on failed lock attempts.

## Notes

`lock` is the primary operation for keeping committed state safe and synchronized.
