## Summary

Enforce safe filesystem handling for extraction, symlink policy, permissions restoration, and plaintext cleanup operations.

## Acceptance Criteria

- Unlock rejects absolute paths and traversal segments, and validates extraction targets remain within allowed root.
- Default behavior does not store or restore symlinks.
- Optional symlink mode is explicit and warns users; portable symlink representation is used.
- POSIX mode persistence/restoration is supported with secure defaults; Windows limitations are documented with best-effort restricted access.
- `--remove` deletes plaintext after successful lock.
- `--shred` is marked best-effort with clear secure-delete limitations.

## Verification

- Run zip-slip and path traversal test cases.
- Validate symlink default-deny and explicit allow behavior.
- Validate permission handling on POSIX and Windows.

## Notes

Safety defaults prioritize plaintext protection over convenience.

This is an umbrella requirement. Atomic security decomposition: `SEC-006`, `SEC-007`, `SEC-008`, `SEC-009`.
