## Summary

`status` and `diff` provide safe introspection in passwordless and authenticated modes, with machine-readable output.

## Acceptance Criteria

- `status` works without password in privacy-preserving summary mode using index/local state semantics.
- With password available, `status` can perform enhanced verification against vault hashes.
- `status` supports `--json`.
- `diff` requires password, provides text diff for text-like files, and binary summaries otherwise.
- `diff` supports `--tool` and respects standard diff/pager environment integration.
- `diff` supports `--json` summary mode.

## Verification

- Validate status behavior in both no-password and authenticated modes.
- Validate diff behavior across text and binary files and tool integration modes.

## Notes

Status privacy constraints are mandatory for default safety posture.
