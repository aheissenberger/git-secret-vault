## Summary

The CLI must provide a version flag that prints the tool name and version, then exits successfully.

## Acceptance Criteria

- Running `git-secret-vault --version` prints the binary name and semantic version and exits with code `0`.
- Running `git-secret-vault -V` behaves identically to `--version`.
- Version output is sourced from package/build metadata (single source of truth) and not duplicated across files.
- Version flag handling is non-interactive and does not require repository state, network access, or secret material.

## Verification

- Validate `cargo run -- --version` prints expected output and exits `0`.
- Validate `cargo run -- -V` output and exit code match `--version`.
- Validate release binary behavior is identical for both flags.

## Notes

This requirement defines externally visible CLI behavior only. Parser/framework implementation details are intentionally left to implementation.
