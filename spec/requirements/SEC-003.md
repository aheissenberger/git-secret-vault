## Summary

Encrypt each blob with XChaCha20-Poly1305 AEAD using a per-blob random nonce; derive vault keys using Argon2id KDF with parameters stored in `vault.meta.json`.

## Acceptance Criteria

- Each blob is encrypted with XChaCha20-Poly1305 AEAD.
- Each encryption operation uses a freshly generated cryptographically random nonce (never deterministic or derived from content).
- Vault key derivation uses Argon2id KDF; parameters (memory, iterations, parallelism, salt) are stored in `vault.meta.json`.
- No AES-256 ZIP encryption is used for authoritative vault storage.

## Verification

- Validate that decrypting a blob with a tampered nonce or ciphertext fails with an authentication error.
- Validate that two encryptions of the same plaintext produce distinct ciphertexts due to random nonce generation.
- Validate that Argon2id KDF parameters are persisted in `vault.meta.json` and loaded correctly on vault open.

## Notes

Replaces AES-256 ZIP encryption as the authoritative crypto suite. XChaCha20-Poly1305 provides a 192-bit nonce space, eliminating nonce-reuse risk when nonces are randomly generated.
