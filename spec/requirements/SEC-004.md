## Summary

Prepend a 24-byte random XChaCha20 nonce to each blob ciphertext; nonces are never reused or derived deterministically.

## Acceptance Criteria

- Each encrypted blob file begins with a 24-byte randomly generated nonce followed by the XChaCha20-Poly1305 ciphertext and 16-byte Poly1305 authentication tag.
- Nonces are generated using a cryptographically secure random source.
- Nonces are never derived from plaintext content, timestamps, counters, or any deterministic input.

## Verification

- Validate blob file structure: first 24 bytes are the nonce, remaining bytes are ciphertext with the 16-byte Poly1305 tag.
- Validate that two encryptions of the same plaintext produce distinct 24-byte nonces.

## Notes

The 24-byte nonce is prepended to the ciphertext in the blob file, enabling stateless decryption without a separate nonce index. The 192-bit nonce space makes random collisions negligible.
