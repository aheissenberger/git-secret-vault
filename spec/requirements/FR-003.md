## Summary

Minimize metadata leakage by default and expose only a minimal non-secret outer index.

## Acceptance Criteria

- Default behavior does not reveal secret filenames or paths without password access.
- Secret content and full manifest stay encrypted inside the vault.
- A configurable outer index file exists (default `.safezipvault.index`) and includes only vault UUID, format version, last-updated timestamp, entry count, and non-secret integrity marker.
- Outer index contains no plaintext filenames or paths.

## Verification

- Inspect index contents and confirm no path/name leakage.
- Validate status/doctor behavior in no-password mode only exposes allowed summary metadata.

## Notes

Filenames becoming visible after successful decrypt/unzip is acceptable and consistent with the format constraint.

This is an umbrella requirement. Atomic security decomposition: `SEC-001`.
