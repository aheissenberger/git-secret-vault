## Summary

`init` creates vault artifacts, establishes identity metadata, and supports secure bootstrap password/keyring flow.

## Acceptance Criteria

- `init` creates vault and index when absent.
- `init` gathers password from secure inputs and confirms interactively when prompted.
- Vault UUID is generated and stored in encrypted manifest and minimal outer index.
- Keyring save is offered as explicit opt-in.
- `--vault` and `--index` path overrides are supported.

## Verification

- Run clean-repo initialization scenarios with interactive and non-interactive password input.
- Confirm generated artifacts and UUID consistency.

## Notes

Initialization must be safe-by-default and idempotent for repeated setup attempts.
