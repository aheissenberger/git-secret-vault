## Summary

Provide repository configuration and local state persistence that support safe automation and privacy-preserving status behavior.

## Acceptance Criteria

- Repo config file supports vault/index paths, include/ignore patterns, conflict defaults, diff tool, password policy, keyring namespace options, and status privacy mode.
- Local-only state file is gitignored and stores opaque per-file fingerprints and lock timing metadata without secrets or plaintext paths.

## Verification

- Validate config parsing and override behavior.
- Validate local state privacy constraints and absence from committed artifacts.

## Notes

Local state exists to enable useful status without password while avoiding metadata leakage.
