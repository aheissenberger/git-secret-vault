## Summary

Provide cargo-dist-based release packaging and publication for GitSecretVault.

## Acceptance Criteria

- Release automation uses `cargo-dist` as the canonical packaging pipeline for supported targets.
- The release workflow publishes `cargo-dist` outputs as versioned release assets.
- Homebrew and other installer metadata generated by `cargo-dist` are published consistently with release artifacts.
- Re-running release automation for the same commit/version is idempotent and does not corrupt published metadata.

## Verification

- Validate `cargo-dist` runs in CI for the configured target matrix and produces expected artifacts.
- Validate GitHub Release assets originate from `cargo-dist` outputs.
- Validate generated install metadata (for example Homebrew formula/tap metadata) matches published artifact checksums.

## Notes

This requirement defines release packaging behavior and tool choice. Exact `cargo-dist` configuration (targets, installers, and publishing strategy) is finalized during implementation.
