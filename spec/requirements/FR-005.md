## Summary

Implement safe password policy, source priority, CI-friendly input modes, and best-effort in-memory secret hygiene.

## Acceptance Criteria

- Configurable password policy is enforced with secure defaults (minimum length, weak-password rejection, passphrase guidance).
- `policy show` and `policy set` expose and update password policy configuration.
- Password source precedence is: `--password-stdin`, explicit env var mode, keyring, interactive prompt.
- Env-var password mode emits leakage warning guidance.
- `--no-prompt` fails fast when no non-interactive password source is available.
- Sensitive buffers are best-effort cleared and limitations are documented.

## Verification

- Validate source priority and `--no-prompt` behavior across interactive and CI paths.
- Confirm policy defaults and configurability.
- Confirm no password material appears in command output.

## Notes

Memory hygiene requirements are best-effort due to language/runtime constraints.

This is an umbrella requirement. Atomic security decomposition: `SEC-002`, `SEC-003`, `SEC-004`, `SEC-005`.
