## Summary

Implement the required GitSecretVault CLI command surface and aliases.

## Acceptance Criteria

- CLI provides: `init`, `lock`, `unlock`, `status`, `ls`, `diff`, `rm`, `passwd`, `keyring`, `verify`, `clean`, `doctor`, `compat check`, and `harden`.
- `ls` is a maintained alias of `status`.
- Command help and usage are available for each required command.

## Verification

- Validate command discovery and help output include all required commands and aliases.

## Notes

Command set parity with lockenv-style workflows is expected.
