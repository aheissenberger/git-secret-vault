## Summary

Define automation-facing output, logging safety, shell UX integrations, CI interfaces, and actionable error contracts.

## Acceptance Criteria

- Query commands (`status`, `verify`, `doctor`, `compat`) support `--json`.
- Default output never prints secret contents or passwords; `--quiet` and `--verbose` are supported.
- CLI integrates with shell UX expectations (completions; `$PAGER`/`$EDITOR` behavior).
- CI modes support `--password-stdin`, `status --fail-if-dirty`, and `lock --check`.
- Error messages include remediation guidance and implement documented exit code mapping.

## Verification

- Validate JSON schema stability for automation commands.
- Validate output redaction/logging safety under normal and verbose modes.
- Validate exit code behavior for key error classes.

## Notes

Automation and operator safety are first-class UX requirements.

This is an umbrella requirement. Atomic decomposition: `SEC-010`, `SEC-011`, `SEC-012`, `NFR-004`, `NFR-005`.
