## Summary

Use a standards-compatible encrypted ZIP vault with built-in CLI crypto operations and explicit compatibility checks.

## Acceptance Criteria

- Vault is written as standards-compatible encrypted ZIP using a broadly supported AES ZIP profile.
- Core encrypt/decrypt operations are implemented internally and do not require external zip/unzip binaries.
- Produced vault is decryptable by common unzip tools that support the selected profile, with compatibility documented.
- `compat check` reports profile/tool compatibility, supports CLI self-test, and warns about likely-incompatible platform default extractors.

## Verification

- Run compatibility and self-test scenarios across supported platforms.
- Confirm vaults created by CLI can be opened with documented external unzip tools.
- Confirm CLI succeeds when external zip/unzip tooling is absent.

## Notes

This requirement codifies the locked decision that unzip compatibility is mandatory while preserving deterministic CLI behavior.
