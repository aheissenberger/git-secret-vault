## Summary

Provide an MCP server interface aligned to MCP 2025-11-25 while preserving CLI as execution core.

## Acceptance Criteria

- MCP server uses JSON-RPC 2.0 and advertises capabilities through MCP-compliant capability negotiation.
- Core operations are exposed as MCP tools with stable input/output schemas for `init`, `lock`, `unlock`, and `status`.
- MCP tool outputs are deterministic and secret-safe, matching `NFR-016` automation constraints.
- Transport-agnostic server behavior supports progress reporting, cancellation, and structured error reporting.
- Security controls follow MCP trust/safety guidance: explicit consent boundaries, tool invocation safety messaging, and least-privilege data exposure.

## Verification

- Execute MCP conformance-style integration tests for initialization handshake and tool invocations.
- Validate schema stability and deterministic output for repeated tool calls.
- Validate cancellation/progress/error pathways and secret-redaction behavior.

## Notes

Recommended architecture pattern: CLI-first core + MCP adapter.

- Keep CLI/library logic as single source of truth for behavior.
- Implement MCP tools as thin adapters that call the same core operations.
- Maintain one shared error taxonomy and one schema contract across CLI JSON and MCP tool results.
