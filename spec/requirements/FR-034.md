## Summary

Provide `vault zip-export` and `vault zip-import` commands that produce and consume AES-256 AE-2 encrypted ZIP files for interchange. ZIP is not authoritative vault storage.

## Acceptance Criteria

- `vault zip-export` produces a ZIP file encrypted with AES-256 AE-2, compatible with standard unzip tools that support that profile.
- `vault zip-import` reads a ZIP produced by `zip-export` and populates the authoritative blob/index structure.
- ZIP export/import operations do not alter or replace the authoritative vault format (blob store + event log).
- Export and import are documented as interchange-only operations; the ZIP is not a substitute for the authoritative vault.

## Verification

- Validate that `vault zip-export` output is decryptable by documented external unzip tools.
- Validate that a `zip-export` → `zip-import` roundtrip preserves all vault entries with no data loss.
- Validate that no ZIP file is written to the vault directory during normal `lock` or `unlock` operations.

## Notes

ZIP interchange preserves compatibility with operators who need to extract secrets using standard tools without the CLI binary present.
