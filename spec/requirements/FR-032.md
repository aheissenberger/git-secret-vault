## Summary

Maintain a canonical snapshot at `index/snapshot.json` sorted by `entry_id`, regenerated by replaying `index/events.jsonl`, for fast vault state recovery without full log replay on every operation.

## Acceptance Criteria

- `index/snapshot.json` represents the current vault state as a JSON object with entries sorted lexicographically by `entry_id`.
- Snapshot is regenerated by replaying all events in `index/events.jsonl` from oldest to newest.
- Snapshot is updated atomically (temp-write + rename) after each vault mutation.
- When `index/snapshot.json` is absent or stale, the vault recovers state by replaying `index/events.jsonl`.

## Verification

- Validate that snapshot entries are sorted lexicographically by `entry_id`.
- Validate that deleting `index/snapshot.json` and reopening the vault produces an identical snapshot via event replay.
- Validate that snapshot writes use a temp-file-then-rename strategy for atomicity.

## Notes

The snapshot is a derived artifact; the event log (FR-031) is the authoritative source of truth. Deterministic sort by `entry_id` ensures identical output across platforms and Go/Rust toolchains.
