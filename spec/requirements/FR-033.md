## Summary

Store vault-level metadata in `vault.meta.json` with fields: `version`, `kdf_params` (Argon2id parameters), `crypto_suite` (identifier string), and `key_ids` (array). The file must not contain plaintext secret paths or filenames.

## Acceptance Criteria

- `vault.meta.json` contains all of the following fields: `version` (semver string), `kdf_params` (Argon2id parameter object including memory, iterations, parallelism, and salt), `crypto_suite` (string, value: `"xchacha20-poly1305"`), `key_ids` (array of key identifier strings).
- `vault.meta.json` contains no plaintext secret filenames, entry names, or cleartext secret material.
- KDF parameters are validated on vault open; a mismatch or unsupported parameter set produces a clear, actionable error.

## Verification

- Validate that `vault.meta.json` produced by `init` contains all required fields with correct types.
- Validate that `vault.meta.json` contains no plaintext secret paths or filenames after `lock`.
- Validate that modifying `kdf_params` in `vault.meta.json` and reopening the vault produces a parameter validation error.

## Notes

`vault.meta.json` is the only unencrypted top-level vault file. It must not leak secret names or paths. All other sensitive data lives in the encrypted blob store.
