## Summary

Maintain an append-only JSONL event log at `index/events.jsonl` recording all vault mutations with required fields: `timestamp`, `op`, `entry_id`, `content_hash`, `key_id`.

## Acceptance Criteria

- `index/events.jsonl` contains one JSON object per line; each line is a complete, self-contained event record.
- Each event record must include: `timestamp` (ISO 8601 string), `op` (one of: `add`, `update`, `remove`, `rotate`), `entry_id` (UUID string), `content_hash` (hex SHA-256 string), `key_id` (string).
- Events are only ever appended; existing lines are never modified or deleted.
- The event log format supports merge compatibility: concurrent appends from independent writers produce a valid log when lines are concatenated.

## Verification

- Validate that each vault mutation (add, update, remove, rotate) appends exactly one event line to `index/events.jsonl`.
- Validate that each event JSON object contains all required fields with correct types.
- Validate that replaying all events in order produces the expected current vault state.

## Notes

The JSONL event log is the authoritative mutation record for the vault. Its append-only, line-delimited design supports distributed merge scenarios and Git-based conflict resolution.
