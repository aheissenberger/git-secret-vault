## Summary

`compat check` must make compatibility risk visible before decryption workflows fail in user environments.

## Acceptance Criteria

- Command reports selected encryption profile and expected compatible tools.
- Optional local self-test mode is supported.
- Command warns when default platform extraction behavior is likely incompatible.

## Verification

- Validate profile reporting and self-test execution across platforms.
